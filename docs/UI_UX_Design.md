# UI/UX Design Concept: WebMaster (Hornygrad)

This document outlines the visual structure and interaction patterns for the web version of Hornygrad. The design follows a "Dark Fantasy" aesthetic with a focus on usability and immersion.

## 1. Main Dashboard (Character Hub)
The central hub where players manage their character and observe their state.

- **Layout:** Three-column design.
    - **Left Column:** Character portrait, Name, Rank, and basic bio.
    - **Center Column:** Main status bars (Essence, Protection) with large, animated counters. Active buffs/debuffs list with timers.
    - **Right Column:** Event log (filtered for character actions) and quick-action buttons (Training, Rest).

## 2. Inventory System
A grid-based interface for item management.

- **Grid:** 10 slots initially (expandable with bags). Items have rarity-colored borders.
- **Weight Indicator:** A progress bar showing current weight vs. max weight. Slowdown penalties are highlighted if overweight.
- **Item Tooltips:** Detailed stats (essence, durability, effects) appearing on hover.
- **Equipment Slots:** Silhouette-based slots for Weapon (Main/Off-hand), Armor (Body), and Artifacts.

## 3. World Map & Navigation
A visual representation of the game world for movement.

- **Global Map:** Simplified topographic map with interactive pins for major locations (Green/Yellow/Red zones).
- **Location View:** When inside a location, shows available buildings (Shop, Inn, Training Ground).
- **Movement:** Clicking a destination initiates a transition with a progress bar (travel time) and occasional flavor text descriptions from the AIService.

## 4. Combat Interface
A turn-based battle screen.

- **Initiative Timeline:** A horizontal bar at the top showing the order of participants.
- **Combat Scene:** Central area showing Character vs. Monster(s) with health bars and action animations.
- **Action Panel:**
    - **Main Action:** Attack, Skill, Item, Move.
    - **Bonus Action:** Use Consumable, Switch Weapon.
- **Battle Log:** A dedicated scrollable area on the side showing every roll and effect application (e.g., "1d100 -> 85: Critical hit!").

## 5. Dialog & LLM Integration
The "Chat-like" interface for interacting with the world.

- **NPC Interaction:** A sliding panel from the right or a modal window.
- **AI Response:** Narrative text generated by the AIService.
- **Player Options:** A mix of predefined choice buttons and a text input for free-form roleplay (future integration).
- **Contextual Awareness:** The AI mentions current location, weather, or previous events to increase immersion.

---

## Technical Considerations
- **Responsive Design:** Mobile-first approach for playing on the go.
- **Offline Mode:** UI elements should clearly indicate when data is being synced to IndexedDB.
- **Animations:** Subtle CSS transitions for state changes (damage flashes, buff applications).
