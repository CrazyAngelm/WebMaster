# 🗺️ Roadmap: Web-версия Hornygrad

Этот документ описывает путь трансформации из Discord-бота в полноценное Web-приложение.

---

## 🎯 Цели проекта
1. **Автономность:** Полный переход на Web-технологии (React + TS).
2. **Оффлайн-first:** Игровая логика работает в браузере (IndexedDB).
3. **Модульность:** Возможность легко добавить мультиплеер и LLM-интеграцию.
4. **Механический стержень:** Реализация всех хардкорных механик боя из оригинальных документов.

---

## 📅 Этапы реализации

### Этап 1: Рефакторинг документации и Архитектура ✅ Завершен
- [x] **Очистка от Discord-зависимостей:** Удалить упоминания ID серверов Discord, префиксов команд и специфичных для бота полей в БД.
- [x] **Адаптация БД:** Перевод схем из `База данных Hornygrad.md` в формат JSON-схем или TypeScript интерфейсов.
- [x] **Проектирование UI/UX:** Наброски интерфейса (Характеристики, Инвентарь, Лог событий, Карта).
- [x] **Выбор архитектуры LLM:** Создание интерфейса-обертки (Mock) для будущей интеграции.

### Этап 2: Ядро игры (Engine) на TypeScript ✅ Завершен
- [x] **Dice System:** Реализация системы бросков (1d100, модификаторы).
- [x] **Character Logic:** Расчет сущности, защиты, скорости и рангов.
- [x] **Battle Loop:** Пошаговый движок боя (Инициатива -> Ход -> Расчет урона -> Конец хода).
- [x] **Inventory System:** Логика веса, стаков, экипировки (броня, оружие, артефакты).

### Этап 3: Frontend-база (React + Vite) ✅ Завершен
- [x] **Setup:** Настройка проекта, Tailwind CSS, Zustand для стейта.
- [x] **Character Sheet:** Визуальное отображение всех статов персонажа.
- [x] **Combat UI:** Экран боя с логом событий (кто кого ударил, сколько прочности снято).
- [x] **Inventory UI:** Сетка предметов с тултипами (описания из `Предметы.md`).

### Этап 4: Хранение данных и Наполнение ✅ Завершен
- [x] **IndexedDB Integration:** Использование Dexie.js для сохранения прогресса игрока.
- [x] **Content Migration:** Перенос всех предметов из `Предметы.md` и механик из `Боевая система.md` в JSON/TS формат.
- [x] **Bestiary Data:** Создание базы монстров и NPC на основе рангов и правил сущности.
- [x] **Static Data:** Загрузка базы предметов, монстров и локаций из статических файлов.
- [x] **Export/Import:** Возможность выгрузить сохранение в файл и загрузить обратно.

### Этап 5: Мир, Прогрессия и Баланс ✅ Завершен
- [x] **Movement:** Перемещение между локациями и зданиями.
- [x] **Events & Quests:** Система триггеров и простых квестов (на базе `Боевая система.md`).
- [x] **Shop & Trade:** Механика торговли с NPC.

### Этап 6: Профессии и Продвинутая Экономика ✅ Завершен
- [x] **Profession System:** Реализация системы профессий (Кузнец, Алхимик и др.) с рангами и накоплением опыта за крафт.
- [x] **Crafting Engine:** Система создания предметов по рецептам с учетом редкости, ингредиентов и рабочих станций (кузница, алхимический стол).
- [x] **Quality Variance:** Логика определения качества созданного предмета через броски кубика (сущность/прочность) согласно рангу профессии.
- [x] **Repair & Maintenance:** Механика износа и починки экипировки с использованием ремонтных наборов различных категорий.

### Этап 7: Хардкорный Бой и Мастеринг Предметов
- [ ] **Advanced Action Economy:** Строгое разделение на основные и побочные действия, реализация механики "времени применения" (Cast Time) для мощных способностей.
- [ ] **Distance & Positioning:** Учет дистанций (Ближняя/Средняя/Дальняя/Снайперская) при атаках и использование перемещения для смены позиции.
- [ ] **Shield & Block Mechanics:** Система блокирования щитом (расход хода, снижение уклонения, поглощение урона прочностью щита).
- [ ] **Item Mastery:** Система "притирки" к оружию (минимальная сущность при покупке, прокачка до максимума через тренировки и бой).
- [ ] **Special Equipment Logic:** Реализация магических стабилизаторов (ячейки заклинаний) и сумок (динамическое расширение слотов инвентаря).

### Этап 8: Наставничество и Высокоуровневая Прогрессия
- [ ] **Mentoring System:** Механика обучения у наставников (передача сущности и опыта профессий между персонажами с разницей в рангах).
- [ ] **Advanced Breakthroughs:** Реализация всех условий прорыва рангов, включая использование специфических зелий и проверку пройденных ивентов нужной редкости.
- [ ] **Combat Maneuvers:** Реализация внеочередных атак при отступлении и механики побега из боя в зависимости от дистанции.
- [ ] **Advanced Effects & Buffs:** Система периодических эффектов (тики кровотечения/отравления) и условий срабатывания (задержка, триггеры).

### Этап 9: Балансировка и Игровой цикл
- [ ] **Balance Tuning:** Проверка и корректировка математики боя, цен в магазинах и силы монстров.
- [ ] **Full Gameplay Loop:** Убедиться, что персонаж может прокачаться с 1 по 6 ранг, посещая все локации и побеждая всех монстров.

### Этап 10: LLM & Отыгрыш (После полной готовности механик)
- [ ] **AI Interface:** Реализация `AIService` (генерация ответов NPC, описание окружения).
- [ ] **Mock Integration:** Работа через заглушку (локальные предопределенные ответы).
- [ ] **Bridge to Backend:** Подготовка API для связи с реальной LLM (OpenAI/Anthropic/Local LLM).

## 🤖 Концепция LLM Mock
Для обеспечения гибкости, интеграция с ИИ будет реализована через абстрактный слой:

```typescript
// 📁 src/services/AIService.ts
export interface AIService {
  generateNPCResponse(npcId: string, playerMessage: string, context: GameContext): Promise<string>;
  describeEvent(event: GameEvent): Promise<string>;
}

// 📁 src/services/MockAIService.ts
// * Текущая реализация (заглушка)
export class MockAIService implements AIService {
  async generateNPCResponse(...) {
    return "Привет! Я пока что имитирую ИИ. Скоро я смогу общаться по-настоящему.";
  }
  // ...
}
```

## 🚀 Финальное состояние
- Полноценная Single-player RPG в браузере.
- Все расчеты автоматизированы (больше не нужно считать пробитие брони вручную).
- Интуитивный интерфейс вместо текстовых команд.
- Готовность к подключению Backend-части для мультиплеера.
