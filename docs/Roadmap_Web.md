# 🗺️ Roadmap: Web-версия Hornygrad

Этот документ описывает путь трансформации из Discord-бота в полноценное Web-приложение.

---

## 🎯 Цели проекта
1. **Автономность:** Полный переход на Web-технологии (React + TS).
2. **Оффлайн-first:** Игровая логика работает в браузере (IndexedDB).
3. **Модульность:** Возможность легко добавить мультиплеер и LLM-интеграцию.
4. **Механический стержень:** Реализация всех хардкорных механик боя из оригинальных документов.

---

## 📅 Этапы реализации

### Этап 1: Рефакторинг документации и Архитектура ✅ Завершен
- [x] **Очистка от Discord-зависимостей:** Удалить упоминания ID серверов Discord, префиксов команд и специфичных для бота полей в БД.
- [x] **Адаптация БД:** Перевод схем из `База данных Hornygrad.md` в формат JSON-схем или TypeScript интерфейсов.
- [x] **Проектирование UI/UX:** Наброски интерфейса (Характеристики, Инвентарь, Лог событий, Карта).
- [x] **Выбор архитектуры LLM:** Создание интерфейса-обертки (Mock) для будущей интеграции.

### Этап 2: Ядро игры (Engine) на TypeScript ✅ Завершен
- [x] **Dice System:** Реализация системы бросков (1d100, модификаторы).
- [x] **Character Logic:** Расчет сущности, защиты, скорости и рангов.
- [x] **Battle Loop:** Пошаговый движок боя (Инициатива -> Ход -> Расчет урона -> Конец хода).
- [x] **Inventory System:** Логика веса, стаков, экипировки (броня, оружие, артефакты).

### Этап 3: Frontend-база (React + Vite) (Текущий)
- [ ] **Setup:** Настройка проекта, Tailwind CSS, Zustand для стейта.
- [ ] **Character Sheet:** Визуальное отображение всех статов персонажа.
- [ ] **Combat UI:** Экран боя с логом событий (кто кого ударил, сколько прочности снято).
- [ ] **Inventory UI:** Сетка предметов с тултипами (описания из `Предметы.md`).

### Этап 4: Хранение данных (Persistance)
- [ ] **IndexedDB Integration:** Использование Dexie.js для сохранения прогресса игрока.
- [ ] **Static Data:** Загрузка базы предметов и локаций из статических JSON файлов.
- [ ] **Export/Import:** Возможность выгрузить сохранение в файл и загрузить обратно.

### Этап 5: Мир и Прогрессия
- [ ] **Movement:** Перемещение между локациями и зданиями.
- [ ] **Events & Quests:** Система триггеров и простых квестов (на базе `Боевая система.md`).
- [ ] **Shop & Trade:** Механика торговли с NPC.

### Этап 6: LLM & Отыгрыш (Mock -> Active)
- [ ] **AI Interface:** Реализация `AIService` (генерация ответов NPC, описание окружения).
- [ ] **Mock Integration:** Работа через заглушку (локальные предопределенные ответы).
- [ ] **Bridge to Backend:** Подготовка API для связи с реальной LLM (OpenAI/Anthropic/Local LLM).

## 🤖 Концепция LLM Mock
Для обеспечения гибкости, интеграция с ИИ будет реализована через абстрактный слой:

```typescript
// 📁 src/services/AIService.ts
export interface AIService {
  generateNPCResponse(npcId: string, playerMessage: string, context: GameContext): Promise<string>;
  describeEvent(event: GameEvent): Promise<string>;
}

// 📁 src/services/MockAIService.ts
// * Текущая реализация (заглушка)
export class MockAIService implements AIService {
  async generateNPCResponse(...) {
    return "Привет! Я пока что имитирую ИИ. Скоро я смогу общаться по-настоящему.";
  }
  // ...
}
```

## 🚀 Финальное состояние
- Полноценная Single-player RPG в браузере.
- Все расчеты автоматизированы (больше не нужно считать пробитие брони вручную).
- Интуитивный интерфейс вместо текстовых команд.
- Готовность к подключению Backend-части для мультиплеера.
