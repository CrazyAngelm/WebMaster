## 🚩 Приоритет 1: Серверный авторитет и безопасность (Завершено ✅)
**Цель:** Исключить возможность читерства и перенести "мозги" игры под контроль сервера.

1.  **Миграция логики боя:** (Выполнено)
    *   Создан серверный аналог `CombatEngine` в `server/src/utils/combatEngine.ts`.
    *   Все расчеты вероятностей, попаданий и урона перенесены на Node.js.
2.  **API для действий:** (Выполнено)
    *   Разработаны защищенные эндпоинты в `server/src/controllers/battleController.ts`.
    *   Эндпоинты принимают `battleId`, `attackerId`, `targetId` и `weaponId`, проверяют авторизацию и ход игрока.
3.  **Валидация ресурсов:** (Выполнено)
    *   Сервер проверяет наличие действий и прав на атаку.
4.  **Стейт-машина боя на сервере:** (Выполнено)
    *   Введены таблицы `Battle` и `BattleParticipant` в Prisma.
    *   Сервер хранит состояние боя, очередь ходов и логи.

## 🚩 Приоритет 1.1: Одномерная боевая система (Завершено ✅)
**Цель:** Реализовать упрощенную одномерную систему боя с управлением расстоянием по оси X.

1.  **Одномерная боевая система (Combat Mode):** (Выполнено)
    *   **Переключение режима**: При инициации боя интерфейс переключается в боевой режим с одномерной линией позиционирования.
    *   **Хранение расстояния**: Добавлено в модель `BattleParticipant` поле `distance` (Float, метры) — расстояние от условного центра боевой линии (0 = центр, положительные значения = правая сторона, отрицательные = левая сторона).
    *   **Визуализация боевой линии**: Горизонтальная линия слева направо, на которой отображаются все участники боя. Расстояние между участниками визуально отражает их `distance` друг от друга.
    *   **Расчет расстояния между участниками**: Для каждой пары участников расстояние = `|distance1 - distance2|`. Чем меньше разница, тем ближе визуально расположены спрайты на экране.
    *   **Действие "Движение"**: Реализованы кнопки "Сближение" и "Отступление", а также клик по линии для точного позиционирования. Движение тратит 1 Main Action.
    *   **Дистанция перемещения**: За одно действие персонаж может изменить `distance` на значение `Speed.distancePerAction` (в метрах).
    *   **Валидация движения**: Сервер проверяет, что изменение `distance` не превышает доступную дистанцию за одно действие. Добавлено минимальное расстояние между персонажами (1.5м) для предотвращения наложения спрайтов.
    *   После окончания боя интерфейс возвращается в режим Exploration с 2D-картой.

2.  **Валидация Дальности атак (Server-side):** (Выполнено)
    *   **Формат хранения**: В `ItemTemplate.distance` поддерживается как JSON строка `{ minRange: number, maxRange: number }`, так и legacy строковые категории (`CLOSE`, `MEDIUM`, `FAR`, `SNIPER`).
    *   Оружие и способности получают `minRange` и `maxRange`.
    *   Сервер перед расчетом `resolveAttack` проверяет условие: `minRange <= distance(A, B) <= maxRange`, где `distance(A, B) = |distanceA - distanceB|`.
    *   Группы дистанций (согласно `docs/Боевая система.md`):
        *   `Melee`: 0–5м.
        *   `Medium`: 5–20м.
        *   `Long`: 20–50м.
        *   `Sniper`: 50–200м.

3.  **Механики сдерживания и побега:** (Выполнено)
    *   **Зона контроля**: Область в радиусе 5м вокруг бойца (в одномерном пространстве: `|distance(A, B)| <= 5`).
    *   **Внеочередная атака (Attack of Opportunity)**: Если персонаж пытается удалиться от врага (увеличить `|distance(A, B)|` более чем на 5м за один ход), враг совершает автоматическую атаку.
    *   **Условие побега**: Кнопка "Сбежать" становится активной только при удалении от всех врагов на расстояние > 90м (`|distance(Player, Enemy)| > 90` для всех врагов).

4.  **Визуализация расстояния в бою:** (Выполнено)
    *   При наведении на врага или выборе действия атаки показывается текущее расстояние до цели и доступные диапазоны атаки текущего оружия.
    *   Визуальная индикация: если цель вне досягаемости, кнопка атаки неактивна или подсвечена красным.
    *   При движении показывается визуальная зона досягаемости (оранжевая полоса) и можно кликнуть по линии для точного позиционирования.

## 🚩 Приоритет 2: Глубина боевой системы и Способности
**Цель:** Довести боевку до соответствия `docs/Боевая система.md`.

1.  **Движок эффектов (Effects & Buffs Engine):**
    *   Создать систему "тиков", которая в начале хода каждого участника обсчитывает активные эффекты.
    *   Добавить поддержку всех категорий:
        *   `Periodic` (DoT/HoT: урон/лечение со временем).
        *   `Blocking` (Stun/Оглушение, блокировка способностей/предметов).
        *   `Weakening/Strengthening` (Buff/Debuff: изменение статов точности, уклонения, урона).
        *   `Control` (Подчинение, страх, берсерк).
    *   Реализовать уровни эффектов (Слабый, Обычный, Сильный) и их влияние на расчеты.
2.  **Система способностей (Skills):**
    *   Реализовать механику "подготовки" (Cast Time), когда способность срабатывает через 1-2 хода после активации.
    *   Добавить систему откатов (Cooldowns), блокирующую повторное использование навыка.
    *   Интеграция с Движком эффектов для наложения баффов/дебаффов.
    *   Валидация редкости способностей относительно ранга персонажа.
3.  **Расходники в бою (Combat Consumables):**
    *   Реализовать использование Зелий и Свитков как Main/Bonus action.
    *   Свитки: расчет урона по формуле `(сущность свитка + сущность посоха)`.
    *   Зелья: применение эффектов на себя/цель/область.
4.  **Расширенные механики боя:**
    *   **Щиты**: Механика блокирования атак с потерей прочности.
    *   **Магические стабилизаторы**: Система ячеек заклинаний для посохов/жезлов.
    *   **Атака двумя оружиями**: Реализация двух бросков на попадание за ход.
    *   **AoE атаки**: Один бросок попадания против нескольких бросков уклонения.
5.  **Система смерти и воскрешения:**
    *   Реализовать статус "Downed" при 0 HP.
    *   Добавить таймер на 3 хода, в течение которых союзники могут "поднять" бойца.
    *   Создать механику перехода в локацию "Загробный мир" при окончательной смерти.

## 🚩 Приоритет 2.1: Прогрессия и Развитие

4.  **Прорывы рангов:**
    *   Автоматизация проверки условий прорыва (ивенты, шансы 1d100, наличие зелий).

2. Все расчеты кубиков подтверждаются сервером.

### Приоритет 1.1: Одномерная боевая система (Завершено ✅)
3. ✅ Участники боя имеют поле `distance` (Float) в `BattleParticipant` для одномерного позиционирования в боевом режиме.
