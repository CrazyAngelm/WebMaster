# 🗺 Дорожная карта реализации WebMaster (Hornygrad Web)

// 📁 docs/Implementation_Roadmap.md - Навигатор реализации
// 🎯 Core function: Пошаговый план превращения прототипа в полноценную Online RPG
// 💡 Usage: Использовать как основной гайд для разработки и приоритизации задач

Этот документ определяет последовательность шагов для достижения полной готовности проекта. Приоритеты расставлены от критически важных (фундамент) к второстепенным (полировка).

---

## 🚩 Приоритет 1: Серверный авторитет и безопасность (Фундамент)
**Цель:** Исключить возможность читерства и перенести "мозги" игры под контроль сервера.

1.  **Миграция логики боя:** 
    *   Создать серверный аналог `CombatEngine`. 
    *   Все расчеты вероятностей, попаданий и урона должны происходить на Node.js. 
    *   Клиент должен получать только итоговый результат анимации и обновленное состояние.
2.  **API для действий:** 
    *   Разработать защищенные эндпоинты (например, `/api/battle/attack`). 
    *   Эндпоинт должен принимать `targetId` и `actionId`, проверять авторизацию игрока и его право на это действие.
3.  **Валидация ресурсов:** 
    *   При каждом действии сервер проверяет: "Достаточно ли у игрока Энергии?", "Есть ли этот предмет в инвентаре?", "Не находится ли игрок слишком далеко?". 
    *   Если проверка не проходит — действие отклоняется с ошибкой.
4.  **Стейт-машина боя на сервере:** 
    *   Ввести таблицу `Battle` и `BattleParticipant` в БД. 
    *   Сервер хранит: чей сейчас ход, какие действия потрачены, текущее здоровье всех участников. 
    *   Это позволит игроку вернуться в бой даже после перезагрузки страницы.

## 🚩 Приоритет 2: Глубина боевой системы и Способности
**Цель:** Довести боевку до соответствия `docs/Боевая система.md`.

1.  **Система навыков (Skills):** 
    *   Реализовать механику "подготовки" (Cast Time), когда способность срабатывает через 1-2 хода после активации. 
    *   Добавить систему откатов (Cooldowns), блокирующую повторное использование навыка.
2.  **Движок эффектов (Effects & Buffs):** 
    *   Создать систему "тиков", которая в начале хода каждого участника обсчитывает активные эффекты. 
    *   Добавить поддержку: DoT (урон со временем), HoT (лечение со временем), Stun (пропуск хода), Buff/Debuff (изменение статов).
3.  **Механика дистанции и движения:** 
    *   Ввести координаты (X, Y) для участников боя. 
    *   Каждое оружие получает параметр `DistanceRange`. 
    *   Добавить действие "Движение", которое тратит Main или Bonus Action для изменения позиции.
4.  **Система смерти и воскрешения:** 
    *   Реализовать статус "Downed" при 0 HP. 
    *   Добавить таймер на 3 хода, в течение которых союзники могут "поднять" бойца. 
    *   Создать механику перехода в локацию "Загробный мир" при окончательной смерти.

## 🚩 Приоритет 3: Real-Time Мультиплеер (Online составляющая)
**Цель:** Сделать мир живым и видимым для всех участников.

1.  **Интеграция Socket.io:** 
    *   Настроить сервер для поддержки WebSocket соединений. 
    *   Каждый игрок при входе подключается к "комнате" своей текущей локации.
2.  **Синхронизация мира:** 
    *   Рассылка события "PlayerMoved" всем в комнате, когда кто-то меняет позицию. 
    *   Рассылка "CharacterUpdated", когда кто-то меняет снаряжение, чтобы другие видели актуальный внешний вид.
3.  **Чат-система:** 
    *   Создать каналы сообщений. 
    *   Реализовать фильтрацию: сообщения локального чата видят только игроки в той же `locationId`.
4.  **Наблюдение за боями:** 
    *   Возможность "подключиться" к ID текущего боя как зритель, получая поток логов и анимаций действий участников.

## 🚩 Приоритет 4: AI Storyteller и Динамический контент
**Цель:** Внедрение LLM для создания уникального игрового опыта.

1.  **Интеграция с LLM API:** 
    *   Создать сервис-прослойку для общения с AI (OpenAI/Anthropic). 
    *   Разработать промпты для оценки текстовых описаний действий (тренировок, крафта, RP).
2.  **Генератор динамических ивентов:** 
    *   Система должна уметь генерировать `GameEvent` на лету, основываясь на тегах локации и истории персонажа. 
    *   Результаты выбора в ивентах должны влиять на БД (выдача предметов, изменение статов).
3.  **Улучшенный NPC-интеллект:** 
    *   Дать NPC "память" о прошлых взаимодействиях с игроком. 
    *   Реализовать ветвящиеся диалоги, где доступность реплик зависит от репутации или ранга.

## 🚩 Приоритет 5: Экономика и Социальное развитие
**Цель:** Создание устойчивого игрового цикла (Core Loop).

1.  **Система магазинов и NPC-торговли:** 
    *   Реализовать инвентарь магазина с ограниченным количеством товара. 
    *   Настроить автоматическое обновление ассортимента (ресток) раз в несколько реальных часов.
2.  **Обмен между игроками (P2P Trade):** 
    *   Создать двухстороннее окно обмена, где оба игрока должны нажать "Подтвердить" после любого изменения состава сделки.
3.  **Групповые активности:** 
    *   Добавить тип локации "Dungeon", вход в который возможен только в группе. 
    *   Реализовать механику "общего лута", распределяемого между участниками.
4.  **Рынок (Аукцион):** 
    *   Создать глобальную таблицу лотов, где игроки могут выставлять предметы на продажу за валюту, даже находясь офлайн.

## 🚩 Приоритет 6: Интерфейс, UX и Баланс
**Цель:** Удобство игры и визуальная привлекательность.

1.  **Адаптивность (Mobile Friendly):** 
    *   Переработать интерфейс инвентаря и чата для удобного управления пальцем. 
    *   Использовать выпадающие меню и модальные окна вместо нагромождения кнопок.
2.  **Визуализация бросков:** 
    *   Интегрировать 3D или 2D анимацию кубиков (например, через Three.js или Canvas). 
    *   Добавить визуальные индикаторы "Критический успех" / "Провал".
3.  **Звуковое сопровождение:** 
    *   Звуки ударов, звон монет, эмбиент для разных типов зон (лес, город, кузница).
4.  **Панель Гейм-Мастера:** 
    *   Инструменты для мониторинга активных игроков, ручного редактирования инвентарей и логирования подозрительной активности.

---

## 📈 Критерии успешности (Definition of Done)
1. Игрок не может изменить свое HP или монеты через консоль.
2. Все расчеты кубиков подтверждаются сервером.
3. Двое игроков в одной комнате видят друг друга и могут торговать/общаться.
4. LLM начисляет бонусы за качественные отыгрыши (Roleplay).
5. Персонаж корректно прогрессирует от 1 до 5 ранга согласно правилам прорыва.
